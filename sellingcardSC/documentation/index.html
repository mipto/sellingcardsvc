General

Note: The top navigation uses Ajax to load pages with transitions. With Ajax, web applications can send data to and retrieve from a server asynchronously (in the background) without interfering with the display and behavior of the existing page. Therefore, you should upload the template to a server to see the transition effects between pages.

Structure

Template folder contains the following folders:

  • assets - contains all .js, .css and *.font files that are related to styling. This folder contains the following sub-folders:

    • css - contains template CSS files
    • less - contains Less files that should be used to create your own skins
    • csslib - contains CSS files of libraries used
    • fonts - contains template font files
    • images - contains template image files
    • js - contains template JavaScript files
    • jslib - contains JavaScript files of libraries and jQuery plugins used
    • svg - contains *.svg files

  • uploads - contains images related to the page content

  • / - root folder which contains HTML page files

HTML

Each *.html file in the root folder represents a separate page.

Each HTML block is wrapped in a comment, which indicates a start and an end of the block. For example:

<!-- .layout-content -->
<div class="layout-content">
...
</div>
<!-- end .layout-content -->

CSS

Template CSS files are located in the assets/css folder.

In this folder you can find the following files:

  • bootstrap-custom.css - CSS files generated based on Bootstrap with the following Less files:

    // Core variables and mixins
@import "variables.less";
@import "mixins.less";

// Reset and dependencies
@import "normalize.less";
@import "print.less";

// Core CSS
@import "scaffolding.less";
@import "grid.less";
@import "buttons.less";

// Components
@import "component-animations.less";

// Utility classes
@import "utilities.less";
@import "responsive-utilities.less";

  • style-panel.css - CSS file for the style panel component. See Style Panel section to learn more about this file.

  • main-skin-style*.css - main CSS file for different skins. You can try each of them via style panel to choose the best fit for you.

Style Panel

The style panel is provided with the template. It allows switching between different color schemes. It can be safely removed by deleting the following pieces of HTML code from each *.html file (which can be found before the closing body tag):

<!-- .style-panel -->
<div class="style-panel" style="display:none">
    <div class="style-panel__button">
        <i class="fa fa-paint-brush"></i>
    </div>
    <div class="style-panel__title">Choose a color:</div>
    <div class="style-panel__colors">
        <a data-value="style1" href="index.html#" class="style-panel__color style-panel__color--style1"></a>
        <a data-value="style2" href="index.html#" class="style-panel__color style-panel__color--style2"></a>
        <a data-value="style3" href="index.html#" class="style-panel__color style-panel__color--style3"></a>
        <a data-value="style4" href="index.html#" class="style-panel__color style-panel__color--style4"></a>
        <a data-value="style5" href="index.html#" class="style-panel__color style-panel__color--style5"></a>
        <a data-value="style6" href="index.html#" class="style-panel__color style-panel__color--style6"></a>
    </div>
    <div class="style-panel__info">and many more...</div>
    <div class="style-panel__reset">
        <button class="btn-reset btn btn-xs btn--style1">reset</button>
    </div>
</div>
<!-- end .style-panel -->
<!-- style panel assets -->
<script type="text/javascript" src="assets/jslib/jquery.cookie.js"></script>
<script type="text/javascript" src="assets/js/StylePanel.js"></script>
<!-- end style panel assets -->

To complete the removal, you can also delete the following files from assets folder:

  • assets/css/style-panel.css
  • assets/js/StylePanel.js
  • assets/jslib/jquery.cookie.js

JavaScript

All functions related to the template functionality are located in the assets/js/Template.js file. This file has a global namespace Template with the following sub-namespaces:

  • Template - contains functions related to the pages and uses other sub-namespaces to implement all required features.
  • Template.menu - contains functions related to the main menu.
  • Template.layout - contains functions related to the layout specific handlers, such as content scrolling and elements placement on the screen.
  • Template.shortcodes - contains functions related to different UI elements.
  • Template.tools - contains different helpers.

Less

We provide a set of Less files that can be used to create your own skin.

Folder assets/less contains the following folders and files:

  • main-variables.less - contains all configurable Less variables.
  • main.less - the core file which includes all others and is used to generate main-skin-*.css files.
  • bootstrap-custom.less - generates bootstrap-custom.css file.
  • layout - contains Less files for layout elements, such as header, footer, sidebar and content area.
  • elements - contains Less files for different UI elements which are used throughout the content (accordion, buttons, rating, social icons, etc.).
  • blog - contains Less files for elements related to the Blog section, such as article information, author information, comments, posts navigation, etc.
  • main-nav - contains Less files for the main navigation menu.
  • page - contains files with different page-specific rules.
  • skin-variables - contains Less files with Less variables which are used to compile main.less file into main-skin-style*.css file.

How to?

How can I create my own skin?

To create your own skin, you should first install Less compiler. Follow the instructions at this link http://lesscss.org/#using-less. It advices how to install a Less processing tool to compile .less file into .css file.

When you have the Less processing tool installed, you should now choose a skin that will be used as basic for the new skin. For example, let’s choose the default one (style5). Now you should create my-skin.less file in assets/less folder with the following content:


@import 'main.less';

@import 'skin-variables/style5.less';

// below I'm making my own adjustments
@main-color:blue;
@main-color2:red;

Now you should go to the assets folder and execute the following command in the console:

:assets$ lessc less/my-skin.less css/my-skin.css

Congratulations! Now you can use assets/css/my-skin.css file as any other predefined Savvy skin.

How can I change the marker location and the icon on the Google map?

To change the marker on the Google map, you should update the configuration object in the file assets/js/Template.js. At the top of the file in the namespace Template you can find the following code:

// configuration object for google map presentation on the contact page
googleMapConfig:{
    // change map marker location here
    markerLocation:{
        lat:-34.397,
        lng:150.644
    },
    // uncomment line below if you want use image or svg file as a marker
    //markerIcon:{ url:'assets/svg/map-marker.svg' },
    zoom:8 // change map zoom value
},

As you see, you can change the marker location, the marker icon and the map zoom level. You can find out more details about marker icons configuration here.

How can I send emails from the contact form to my email address?

Savvy is an HTML template so no back-end scripts are added by default. However, we supply a simple PHP script that you can use to receive email notifications from the contact form.

NOTE: please ensure that your hosting has PHP support and a working mail function. You can find more about the mail function here.

So if you have PHP on your server and mail function is working, you should make following changes:

  • Complete email notification config in the file mail/mail.php. For this, you should update constants at the top of this script:
// configurate notification subject and email address here
// to change notification message please see 'fetchNotificationBody' function
define('NOTIFICATION_SUBJECT', 'Savvy: Contact form notification');
define('NOTIFICATION_EMAIL_ADDRESS', ''); // for example: 'your.email@host.com'
  • You should change action attribute in the contact form to use mail/mail.php script instead of mail/fake.js. Open file contact.html and find the following lines:
<!-- #contactForm -->
<form action="mail/fake.js" id="contactForm" method="post" class="layout-horizontal-padding">

and replace action attribute value, so that it looks like this:

<!-- #contactForm -->
<form action="mail/mail.php" id="contactForm" method="post" class="layout-horizontal-padding">
  • And now try to send a message!
